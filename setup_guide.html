Shopify App Development Setup Guide

Active Configuration

1. Current Stack Architecture

FrameworkReact Router v7
Uses the modern React Router Single-Page App Framework with Vite bundling and type generation support.
DatabasePrisma & SQLite
SQLite local database (prisma/dev.sqlite) handles active pools and member tracking.
UI SystemShopify Polaris
Polaris components wrapped inside a root AppProvider to match the Shopify admin design tokens.

2. Implemented Fixes

Fix 1Resolved useContext / Duplicate Context Error
Uninstalled duplicate react-router-dom dependency. Updated vite.config.ts with optimizeDeps.include settings for react, react-dom, and react-router to enforce singleton resolution in development.
Fix 2Resolved Sibling Route Mapping (Blank Screen)
Renamed app.pools.tsx to app.pools._index.tsx. This prevents React Router from interpreting the list page as a layout for /app/pools/new, allowing sibling routes to render correctly inside the /app layout.
Fix 3Shopify App Bridge Session Loader
Added authentication loaders to app.pools.new.tsx and app.additional.tsx to guarantee that App Bridge security and CSP headers are written for every page accessed.

3. Local Development Run Guide

To run the app locally and test changes in the Shopify development store:

# 1. Generate client and run database migrations
npx prisma generate
npx prisma migrate dev --name init

# 2. Run the Shopify app CLI dev server
npm run dev

Access the app dashboard via the stable Shopify Admin link:

https://admin.shopify.com/store/zourcefy-dev/apps/zourcefy-group-buy/app/pools
